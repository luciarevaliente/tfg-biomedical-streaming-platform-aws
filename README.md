## 📋 Overview

Cloud platform for the **ingestion and near real-time processing of biomedical data** generated by wearable devices, designed for continuous patient monitoring in non-critical hospital areas.

The system receives continuous streams of physiological signals (ECG, EDA, BVP, TEMP, RESP...), processes them with end-to-end latencies in the order of seconds, and stores both raw and processed data for traceability and analysis.

---

## 🏗️ Architecture

The platform follows a **fully serverless, event-driven architecture** on AWS, structured in five functional blocks:

```
Data Generator → Ingestion → Processing → Persistence → Monitoring
   (local)        Kinesis      Lambda       S3 + DynamoDB   CloudWatch
```

| Component | AWS Service |
|---|---|
| Ingestion | Amazon Kinesis Data Streams |
| Processing | AWS Lambda + SQS DLQ |
| Raw persistence | Amazon S3 |
| Processed persistence | Amazon DynamoDB |
| Monitoring & audit | Amazon CloudWatch + AWS CloudTrail |
| Infrastructure | Terraform |

**Target SLO:** P95 end-to-end pipeline latency < 10 seconds.

## 📡 Data Model

The system uses the **WESAD dataset** (Wearable Stress and Affect Detection) as a structural reference to define the schema and sampling frequencies of the simulated events.

### Sensors and sampling frequencies

| Device | Sensor | Signal | Frequency |
|---|---|---|---|
| RespiBAN | ECG, EDA, EMG, TEMP, RESP, ACC | Various | 700 Hz |
| Empatica E4 | BVP | Pulse volume | 64 Hz |
| Empatica E4 | ACC | Acceleration | 32 Hz |
| Empatica E4 | EDA, TEMP | Electrodermal, Temperature | 4 Hz |

### Record types

- **Raw record** — event as ingested into the system (subject, device, sensor, value, timestamps...)
- **Processed record** — enriched with processing timestamp, validity flag and pipeline latency

## 🚀 Deployment

### Prerequisites

- AWS CLI configured (`aws configure`)
- Terraform installed
- AWS account with IAM user with `AdministratorAccess`

### Deploy infrastructure

```bash
cd infrastructure/terraform

# Initialize Terraform with S3 backend
terraform init -backend-config=backend.hcl

# Preview changes
terraform plan -var-file=terraform.tfvars

# Apply
terraform apply -var-file=terraform.tfvars
```

### Destroy infrastructure (to avoid costs)

```bash
terraform destroy -var-file=terraform.tfvars
```
